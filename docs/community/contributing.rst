=================================
Contributing to GoFast
=================================

We welcome contributions to the GoFast project! This document will guide you through the process of contributing to the GoFast library.

.. toctree::
   :maxdepth: 2

   reporting_issues
   submitting_pull_requests
   coding_standards
   community_guidelines

Reporting Issues
----------------

Clear, detailed reports are invaluable in helping us identify and fix issues.

.. include:: reporting_issues.rst

Guidelines for Reporting Issues:

- Use the GitHub issue tracker: `https://github.com/WEgeophysics/gofast/issues`
- Check existing issues to avoid duplicates.
- Include detailed information: your environment, GoFast version, steps to reproduce, and any relevant code snippets.

Submitting Pull Requests
------------------------

We appreciate pull requests. Here's a quick guide:

.. include:: submitting_pull_requests.rst

- Fork the repository and create your branch from `master`.
- Write clear, concise commit messages.
- Include tests for new functionality.
- Ensure your changes pass all tests.
- Update the documentation as necessary.
- Submit a pull request through GitHub.

Coding Standards
----------------

Consistent coding standards improve the quality and maintainability of the codebase.

.. include:: coding_standards.rst

- Follow the PEP8 style guide for Python code.
- Write clear docstrings for all public modules, functions, classes, and methods.
- Comment your code where necessary.
- Use descriptive variable names and keep functions focused and compact.

Community Guidelines
--------------------

Our community abides by a code of conduct that everyone is expected to follow.

.. include:: community_guidelines.rst

- Be respectful and inclusive.
- Offer constructive feedback.
- Acknowledge and credit contributions from others.
- Report inappropriate behavior to the project maintainers.
